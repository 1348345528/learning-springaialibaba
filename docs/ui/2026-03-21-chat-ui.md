# 智能问答模块 UI 设计文档

## 1. 概述

### 1.1 模块简介
智能问答模块是 RAG 知识库系统的核心功能，允许用户输入自然语言问题，从知识库中检索相关内容，并生成 AI 回答。

### 1.2 技术栈
- React 18.2.0
- Vite 5.0.8
- Ant Design 5.12.0
- SSE (Server-Sent Events) 流式响应

### 1.3 设计目标
- 提供清晰的用户交互界面
- 支持流式回答的实时展示
- 区分用户与 AI 的视觉样式
- 适配不同屏幕尺寸

## 3. 组件清单

### 3.1 ChatPage (页面容器)
| 属性 | 类型 | 说明 |
|------|------|------|
| - | - | 包含所有聊天子组件，管理全局状态 |

### 3.2 ConversationList (对话列表)
| 属性 | 类型 | 说明 |
|------|------|------|
| messages | Message[] | 消息列表 |
| loading | boolean | 是否正在加载 |
| streamingId | string \| null | 当前流式输出的消息ID |

### 3.3 MessageBubble (消息气泡)
| 属性 | 类型 | 说明 |
|------|------|------|
| message | Message | 消息对象 |
| isStreaming | boolean | 是否正在流式输出 |
| onCopy | () => void | 复制回调 |
| onToggleExpand | () => void | 折叠/展开回调 |

### 3.4 InputArea (输入区域)
| 属性 | 类型 | 说明 |
|------|------|------|
| onSend | (text: string) => void | 发送回调 |
| onReset | () => void | 重置回调 |
| disabled | boolean | 是否禁用输入 |
| maxLength | number | 最大字数 (默认500) |

## 4. 消息数据结构

```typescript
interface Message {
  id: string;              // 唯一标识
  role: 'user' | 'assistant';  // 角色
  content: string;         // 内容
  timestamp: number;       // 时间戳
  status?: 'loading' | 'error' | 'success';  // 状态
}
```

## 8. 交互流程

### 8.1 发送消息流程
```
1. 用户在输入框输入问题
2. 输入框实时显示字数 (0/500)
3. 用户点击"发送"按钮 或 按 Enter 键
4. 输入框禁用，按钮变为 loading 状态
5. 用户消息立即显示在对话列表（气泡+头像+时间戳）
6. AI 消息气泡出现，显示 loading 动画
7. 流式内容逐段追加到 AI 消息气泡
8. 流式结束，移除 loading 状态，显示复制/折叠按钮
9. 输入框恢复可用
```

### 8.2 重置流程
```
1. 用户点击"重置"按钮
2. 弹出确认 Modal: "确认清空对话记录？"
3. 用户确认后，清空所有消息
4. 输入框恢复可用
```

### 8.3 复制流程
```
1. 用户点击"复制"按钮
2. 复制消息内容到剪贴板
3. 显示 Ant Design message 提示: "已复制到剪贴板"
```

### 8.4 折叠/展开流程
```
1. 当 AI 回答超过 5 行时，默认折叠
2. 显示"展开"按钮
3. 用户点击"展开"，显示完整内容，按钮变为"折叠"
4. 用户点击"折叠"，恢复折叠状态
```

## 9. 状态设计

### 9.1 加载状态
- **全局加载**: 页面首次加载时，中心显示 Spin 组件
- **消息加载**: AI 回答区域显示脉冲动画的占位文字
- **流式加载**: AI 回答区域显示闪烁光标 (|)

### 9.2 空状态
- 对话列表为空时显示:
  - 图标: RobotOutlined (antd icons)
  - 文字: "还没有对话记录，请开始提问吧"
  - 样式: 居中显示，颜色 #bfbfbf

### 9.3 错误状态
- **网络异常**:
  - AI 消息气泡显示红色边框
  - 文字: "抱歉，网络连接失败，请重试"
  - 显示"重试"按钮

- **API 错误**:
  - AI 消息气泡显示错误信息
  - 文字: "抱歉，服务端出错: {error message}"
  - 显示"重试"按钮

### 9.4 输入框状态
| 状态 | 样式 |
|------|------|
| 正常 | 白底，#d9d9d9 边框 |
| 聚焦 | 白底，#1677ff 边框，浅蓝色阴影 |
| 禁用 | #f5f5f5 背景，#bfbfbf 文字 |
| 字数警告 | 输入框边框变为 #faad14 |
| 字数超限 | 输入框边框变为 #ff4d4f，无法输入 |

## 10. 响应式设计

### 10.1 断点定义
| 设备 | 最小宽度 | 最大宽度 |
|------|----------|----------|
| 移动端 | - | 767px |
| 平板端 | 768px | 1023px |
| 桌面端 | 1024px | - |

### 10.2 响应式规则
| 元素 | 移动端 | 平板端 | 桌面端 |
|------|--------|--------|--------|
| 消息气泡最大宽度 | 95% | 85% | 80% |
| 输入框宽度 | 100% | 100% | calc(100% - 180px) |
| 侧边栏 | 收起 | 展开 | 展开 |
| 头像大小 | 32px | 36px | 40px |

### 10.3 移动端适配
- 侧边栏默认收起，只显示图标
- 点击图标展开菜单抽屉 (Drawer)
- 输入区域固定在底部
- 消息气泡占满屏幕宽度

## 12. API 接口 (已有，无需修改)

### 12.1 流式聊天接口
```javascript
chatApi.streamChat(
  { question: "用户问题" },
  onMessage,   // 收到消息的回调
  onError      // 错误回调
)
```

### 12.2 SSE 消息格式
```javascript
// 用户消息
{ type: 'user', content: '问题内容', timestamp: 1234567890 }

{ type: 'chunk', content: '回答片段' }
```

## 13. 可访问性 (A11y)

### 13.1 键盘导航
- Tab 键: 在输入框和按钮间切换
- Enter 键: 发送消息
- Escape 键: 取消输入

### 13.2 ARIA 标签
- 输入框: `aria-label="问题输入框"`
- 发送按钮: `aria-label="发送消息"`
- 重置按钮: `aria-label="清空对话"`
- 消息列表: `role="log" aria-live="polite"`

### 13.3 对比度
- 文字与背景对比度 >= 4.5:1
- 大文字 (>=18px) 对比度 >= 3:1

## 14. 性能考虑

### 14.1 虚拟滚动
- 当对话消息超过 100 条时，使用虚拟滚动
- 只渲染可视区域内的消息

### 14.2 防抖
- 输入框字数统计防抖: 150ms
- 自动保存草稿防抖: 1000ms

### 14.3 懒加载
- 对话列表懒加载历史消息
- 预加载下一条回答所需资源
